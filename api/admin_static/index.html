Getting Started

Start the proxy server and open the admin dashboard.

1
Start the server

Open a terminal in your project directory and run:

fcc-server

Or, using the dev virtualenv:

uv run fcc-server
2
Open this dashboard

Navigate to:

http://127.0.0.1:8082/admin

The admin dashboard is local-only — it only responds to requests from 127.0.0.1 / localhost.

3
Point your AI tool at the proxy

Set the API base URL of Claude Code, OpenCode, or any OpenAI-compatible tool to:

http://127.0.0.1:8082

Use any non-empty string as your API key — the proxy supplies real credentials from its rotation pool.

Providers View

Configure and health-check upstream AI providers.

1
Click Providers in the sidebar. Each card shows a live status pill: Healthy, Warn, or Offline.
2
Click Ping on any provider card to test live latency and refresh its status.
3
Edit credentials (API keys, base URLs) directly in the text inputs, then click Apply in the bottom bar.
4
Click Explore Models to see all models available from a provider (queries the provider's /models endpoint).

Model Config View

Control routing, thinking mode, and web tools.

1
Set the Primary Model and Fallback Model from the dropdowns (or type a custom model ID).
2
Enable Extended Thinking and set the maximum thinking token budget.
3
Toggle Web Tools (search, page fetch) on or off for all proxied requests.
4
Click Show Diff to preview your changes, then Apply to commit them to the config file.

Analytics & History View

Monitor every request routed through the proxy — latency, cost, errors, and provider splits.

1
Summary cards (top row) update on each visit:
CardDescription
Total RequestsAll-time count of requests handled by the proxy
Total Cost (USD)Estimated LLM token cost in USD
Avg LatencyMean end-to-end latency across all requests
Error Rate% of requests that returned a 4xx / 5xx status
2
Provider Distribution — sorted progress bars show each provider's traffic share, plus a stat strip with p50 / p95 / p99, TTFB, Fallbacks, and Streaming %.
3
Request History Log — newest requests first. Columns: Timestamp · Request ID · Provider / Model · Tokens (I/O) · Latency (TTFB / Total) · Status code. An amber Fallback badge marks cascade requests.
4
Click Export to CSV to download a full fcc_analytics.csv with all request fields.

Live Logs View

Real-time server log stream in the browser.

1
Click Live Logs — the terminal auto-connects via SSE to /admin/api/logs/stream.
2
Logs appear in real-time as requests are processed. Click Clear to wipe the scrollback buffer. Navigating away automatically closes the stream.

Comparison & Explorer View

Benchmark all providers side-by-side.

1
Click Comparison & Explorer — the table shows last-known status, latency, and model count for every configured provider.
2
Click Run Benchmark to ping all providers in sequence. Use the row-level Ping button to test a single provider.

Profiles & Config

Multi-environment profiles and config import/export.

1
Use the Profile Selector (top-right sidebar) to switch between .env profiles (e.g. default, dev, prod). Click + to create a new profile.
2
Export (bottom bar) — downloads your current configuration as a JSON file for backup or sharing.
3
Import (bottom bar) — loads values from a previously exported JSON file. Click Apply after importing to persist the changes.
4
Click Light Mode / Dark Mode (top-right) to toggle the theme. Your preference is saved in localStorage.
